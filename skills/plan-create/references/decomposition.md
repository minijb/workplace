# 任务分解策略（plan-create 参考）

本文件给 `plan-create` skill §3 提供分解方法。**分解任务前先 `read` 本文件。**

## 核心原则：垂直切片，不是水平铺层

**坏的（水平切片）**——按技术层铺，最后才连：

```
T1: 建全部数据库 schema
T2: 建全部 API 端点
T3: 建全部 UI 组件
T4: 把它们连起来
```

问题：T1-T3 完成时没有任何可测功能；T4 是隐藏的集成炸弹；前置错误到 T4 才暴露。

**好的（垂直切片）**——每个任务端到端打通一条可测路径：

```
T1: 用户能注册（schema + API + UI）
T2: 用户能登录（auth schema + API + UI）
T3: 用户能创建任务（task schema + API + UI）
T4: 用户能看任务列表（query + API + UI）
```

每个切片交付**工作、可测**的功能。集成风险被分散到每个任务里持续消化。

## MECE：相互独立、完全穷尽

- **相互独立**：两个任务不应做同一件事。若重叠，合并或划清边界。
- **完全穷尽**：合起来必须能达成 `goal`。自检：完成全部任务后，`acceptance`（计划级）是否必然满足？若否，缺任务——补上。

判定技巧：把任务标题列出来，逐对问"这俩是同一件事吗？"，再问"做完这些，目标达成了吗？"。

## 任务尺寸标准

| 尺寸 | 触碰文件 | 范围 | 处置 |
|---|---|---|---|
| XS | 1 | 单函数或配置改 | 直接做 |
| S | 1-2 | 一个组件或端点 | 直接做 |
| M | 3-5 | 一个功能切片 | 直接做 |
| L | 5-8 | 多组件功能 | **拆** |
| XL | 8+ | — | **必须拆** |

executor 在单任务上表现最好的是 S-M。L 及以上**必须继续拆**。

**继续拆的信号**：

- 标题里出现"和"/"以及"——多半是两个任务。
- 触碰 ≥2 个相互独立的子系统（如 auth 和 billing）。
- acceptance 写不出 ≤3 条具体可测条件。
- 估算 executor 要跨多次专注会话才能完成。

## 每个任务必备的字段

```md
### T2: 实现 JwtSessionProvider
- status: todo
- mode: subagent              # 可选，缺省继承 default_mode
- depends: [T1]
- acceptance:                 # 具体可测，回答"做对了吗"
  - JwtSessionProvider 通过 SessionProvider 契约的全部测试
  - token 过期后端点返回 401
- files: [src/auth/JwtSession.ts]
- verification: pnpm test src/auth/jwt   # 可执行命令
```

- **acceptance**：任务级，回答"做对了吗"。每条要能跑命令或能点开验证。
- **verification**：可执行命令或可观察检查。executor 完成任务后跑它确认。
- **depends**：任务 id 列表。executor 不会在前置未全 `done` 时调度本任务。
- **files**：预计触碰面。executor 用它做冲突检测（subagent 模式下尤其重要）。

## 依赖排序

实现序 = 依赖图的拓扑序。规则：

1. **地基先行**：被依赖最多的底层抽象（schema、契约接口）先做。
2. **每个任务后系统仍可工作**：不要产生"做完 T2 系统就不能编译"的中间态——要么 T2 自包含，要么把它拆小。
3. **高风险前置**：不确定的技术选型、外部依赖、新接口——尽早做，失败早暴露。
4. **检查点每 2-4 任务**：列出该阶段必须通过的验证。检查点是 executor 的暂停点。

## 何时不要再拆

过度分解同样有害——任务太碎则依赖图爆炸、协调开销压过实现。判定：

- 任务已是 S-M 尺寸。
- acceptance 清晰可测。
- 与其他任务边界明确。
- 进一步拆只会产生"建文件""导出符号"这种本身无价值的碎片。

满足全部即可停止分解。

## 反模式

| 反模式 | 表现 | 修正 |
|---|---|---|
| 水平铺层 | "建全部 X，再建全部 Y" | 改垂直切片 |
| 标题含"和" | "实现登录和注册" | 拆成两个任务 |
| 无 acceptance | "实现该功能" | 写出可测条件 |
| 无 verification | 只说"测试通过" | 给出具体命令 |
| 悬空依赖 | depends 引用不存在的 id | 修引用或补任务 |
| XL 任务 | 触碰 8+ 文件 | 继续拆 |
| 隐藏集成 | 最后一个任务叫"连接一切" | 集成风险分散到每个切片 |
