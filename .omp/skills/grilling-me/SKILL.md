---
name: grilling-me
description: 针对一份计划或设计方案，对用户进行毫不留情的盘问。当用户想在动工前压力测试某个计划，或使用任何 'grill' 触发词时使用。
---

本 skill 只做盘问、不落盘、不维护 docs/ADR；若需把盘问中结晶的术语、决策、未决冲突沉淀进 docs 系统，转用 `grilling-with-context`。

就这份计划的每个方面对我进行毫不松懈的盘问，直到我们达成共识。逐一走完设计树（设计树＝该计划涉及的所有决策维度及其依赖关系）的每个分支，一个一个地理解决策之间的依赖关系。对每个问题，都给出你推荐的答案。在提问之前先查找 docs 中是否含有对应知识，通过索引获取可能需要的知识。

每次只问一个问题，每问完一个就等我的反馈再继续。一次性抛出多个问题会让人无所适从。若某分支我坚持不答，标注为未决并继续下一分支，不要卡死。

如果一个问题可以通过探查代码库来回答，那就去探查代码库，而不是来问我。

在我确认我们已达成共识之前，不要开始执行这份计划。

当你判断设计树已走遍或我确认共识时，给出本次盘问的共识摘要（已决项 + 仍悬而未决项），然后把"下一步"交还给我——本 skill 不负责实现或规划。

节奏示例（1 问 + 推荐答案 + 等反馈）：
> 「认证用 JWT 还是 session？我推荐 JWT——无状态、易水平扩展。你的判断？」问完即停，等反馈，再进下一题。
